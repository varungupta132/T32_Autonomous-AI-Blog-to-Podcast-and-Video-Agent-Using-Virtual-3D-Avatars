# Contributing to AI Blog-to-Podcast Generator

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/T32_Autonomous-AI-Blog-to-Podcast-and-Video-Agent-Using-Virtual-3D-Avatars.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`

## Development Setup

1. Install Python 3.8 or higher
2. Install Ollama from https://ollama.com/download
3. Install dependencies: `pip install -r requirements.txt`
4. Pull an Ollama model: `ollama pull llama2`

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and single-purpose

## Testing

- Test your changes locally before submitting
- Ensure all existing functionality still works
- Add tests for new features when applicable

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove)
- Keep the first line under 50 characters
- Add detailed description if needed

Examples:
- `Add error handling for missing audio files`
- `Fix speaker label normalization bug`
- `Update README with installation instructions`

## Pull Request Process

1. Update documentation if needed
2. Test your changes thoroughly
3. Create a pull request with a clear description
4. Link any related issues

## Areas for Contribution

- Bug fixes
- Documentation improvements
- New TTS engine integrations
- UI/UX enhancements
- Performance optimizations
- Test coverage

## Questions?

Open an issue for discussion before starting major changes.
